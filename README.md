## About Me

- Breaking things in dev so users don't have to in prod
- Full-stack chameleon — frontend, backend, databases, whatever it takes to ship
- Currently running experiments that have a 60% chance of working (optimistically)
- If it's repetitive, it's getting automated. No exceptions. Not even this bullet point.
- Learning philosophy: stack overflow first, docs second, actually read the code third
- Open-source enthusiast — if it's free and it works, I've probably already forked it
- Debugging at 2AM so you can use a tool that works at 9AM
- Obsessed with building tools that make people go "wait, this actually works?"
- Powered by Python, caffeine, and the audacity to think this will compile
- End goal: build things so useful that people forget I made them
